# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

A single-file browser-based Connect 4 game. No build step, no dependencies, no package manager — open `connect4.html` directly in a browser to run.

## Architecture

Everything lives in `connect4.html` as a self-contained file with three sections:

- **CSS** (`<style>`): Styles the dark-themed board, animated drop pieces, hover preview row, win pulse animation, and scoreboard.
- **HTML** (`<body>`): Declares the static scaffold — `#status`, `#preview-row`, `#board`, `#controls`, and `#scores`. The board cells themselves are generated by JavaScript.
- **JavaScript** (`<script>`): All game logic. Key globals and functions:
  - `board` — 6×7 2D array (`null | 'red' | 'yellow'`)
  - `currentPlayer`, `gameOver`, `scores` — mutable state
  - `init()` — resets state and re-renders
  - `render()` — rebuilds the entire board DOM and preview row from scratch on every move
  - `drop(col)` — handles a move: finds lowest empty row, updates `board`, calls `render()`, triggers drop animation, then calls `checkWin()`
  - `checkWin(row, col)` — checks 4 directions from the last placed piece; returns the winning cell array or `null`
  - `highlightWin(cells)` — adds `.win` class (pulse animation) to winning pieces

## Git Workflow

Remote: `https://github.com/jayjaythejet/ClaudeCodeTest`

**Commit and push after every meaningful unit of completed work** — a feature added, a bug fixed, a refactor done. Never leave the repo in a state where progress could be lost. Do not batch unrelated changes into one commit.

Commit message format: imperative mood, concise subject line that describes what changed and why.
- Good: `Add drop animation duration scaling by row depth`
- Good: `Fix checkWin not detecting diagonal wins at board edge`
- Avoid: `updates`, `fix stuff`, `WIP`

```bash
git add <files>
git commit -m "concise description of what and why"
git push
```
